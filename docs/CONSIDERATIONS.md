# 仕様・考慮事項

実装時の前提と、既知の挙動・運用上の注意点です。

## 対象年齢

| 項目 | 内容 |
|------|------|
| 推奨 | **0〜6歳**向けにコンテンツ・AIを最適化 |
| 利用 | **7歳以上**のお子さんがいても登録・相談は可能（注意表示のみ） |
| 登録 | 未来の誕生日のみ不可。それ以外の年齢は保存可 |
| AI | 0〜6歳を優先しつつ、それ以外の年齢も可能な範囲で支援 |

## プラン・課金

| 項目 | 内容 |
|------|------|
| プラン変更 | マイページ「お支払い管理」→ Stripe Customer Portal（要ダッシュボード設定） |
| 解約 | 同上。`customer.subscription.deleted` で Free に自動反映 |
| 支払い失敗 | リトライ中は Free にしない。`past_due` / 解約完了後に Free |
| Checkout | 既存 `stripe_customer_id` があれば同一 Customer に紐付け |
| `plan` 列 | クライアントから更新不可。Webhook / service_role のみ |

## 子ども・関係性（Pro）

| 項目 | 内容 |
|------|------|
| 複数登録 | Pro のみ（DB トリガーで Free/Lite は1人） |
| 関係の保存 | 「登録しない」は DB に行を作らない |
| 年上・年下 | 誕生日（と性別）から自動判定。フォームは種類のみ |
| 性別未回答 | 関係保存時は `other` として兄/弟側にフォールバック |
| 関係の置換保存 | ある子の保存時、その子が関わる行を一度削除してから再挿入 |
| 子ども削除 | 最後の1人は不可（DB トリガー + UI） |
| Pro→Free 後 | 複数子どもデータは残る。相談は1人を選択して利用 |

## チャット

| 項目 | 内容 |
|------|------|
| 履歴の順序 | `created_at` 昇順、同一時刻は user → assistant |
| 保存 | Lite/Pro は user / assistant を別 INSERT（順序保証） |
| Free 制限 | 送信前に user メッセージを DB 挿入してから件数カウント |
| 履歴日数 | Free: 7日、Lite/Pro: 無制限 |

## マイページ UX

| 項目 | 内容 |
|------|------|
| 編集から戻る | `router.back()` + 必要時のみ子どもデータをクライアント再取得 |
| 相談切替 | カード右上（相談中ラベルと同位置） |

## 未対応・将来検討

- ポータル未設定時のエラーメッセージの Stripe 原文表示
- 解約後も複数子どもを自動削除しない（データ保持方針）
- E2E / API 統合テスト（現状は純粋関数のユニットテストのみ）
