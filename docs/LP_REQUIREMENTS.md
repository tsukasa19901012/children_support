# LP（ランディングページ）要件定義 — となりっこ

**版:** 1.1  
**対象:** `https://www.tonarikko.com` 配下の公開ページ（アプリ本体とは別導線の「説明・法務・問い合わせ」）  
**正:** 表示文言・プランは `src/lib/brand.ts` / `src/features/billing/plans.ts` と一致させる

---

## 1. 目的

| # | 目的 |
|---|------|
| 1 | 初見ユーザーにサービス価値を **30秒で理解** させる |
| 2 | **利用規約・プライバシー・特商法・お問い合わせ** を恒久的な URL で公開する |
| 3 | アプリ（ログイン後）・Stripe・メールから **同じ URL を参照** できるようにする |
| 4 | 不安を煽らず、**落ち着いたトーン** で信頼を得る（RULES.md / docs/ux.md 準拠） |

**非目的（LP ではやらない）**

- SNS・口コミ・ゲーム要素
- 医療診断の提供（AI は育児支援であり医療ではない旨は明記）
- ログイン必須の機能説明の代替（詳細操作はアプリ内）

## 2. 想定ユーザー

| 区分 | 説明 |
|------|------|
| プライマリ | 0〜6歳の子を育てる保護者（疲れていて、短く相談したい） |
| セカンダリ | 7歳以上も利用可だが、LP では **0〜6歳向け** を前面に |
| リファラー | 検索・紹介・Stripe Checkout からの戻り・メールフッター |

## 3. URL 構成（推奨）

同一ドメイン・同一 Next.js アプリ内の静的/SSR ページとして実装する想定。

| パス | ページ名 | 必須 |
|------|----------|------|
| `/` | トップ（サービス LP） | ○（現状はチャット。方針は **後述 7.1**） |
| `/lp` または `/about` | サービス紹介 LP（未ログイン向け） | ○（トップをチャットのままなら必須） |
| `/terms` | 利用規約 | ○ |
| `/privacy` | プライバシーポリシー | ○ |
| `/legal` | 特定商取引法に基づく表記 | ○（有料 Plus があるため） |
| `/contact` | お問い合わせ・サポート | ○ |

**命名方針**

- 英語パスで統一（`/terms` 等）。日本語スラッグは避ける（メール・Stripe で貼りやすい URL）
- すべて `https://www.tonarikko.com`（**www 必須**）

**フッター共通リンク（全 LP・法務ページ）**

```
利用規約 | プライバシーポリシー | 特定商取引法に基づく表記 | お問い合わせ
```

## 4. ページ別要件

### 4.1 サービス紹介 LP（`/lp` 等）

#### 4.1.1 情報設計（セクション順）

1. **ヒーロー**
   - サービス名: となりっこ
   - サブ: うちの子を覚える育児AI
   - タグライン: `BRAND.tagline`（うちの子のこと、覚えながら聞く。）
   - 補足: `BRAND.subtagline`（0〜6歳の育児の話、落ち着いて。）
   - CTA 主: **無料ではじめる** → `/login`
   - CTA 副: 機能を見る（ページ内アンカー）

2. **共感（悩み）**
   - 例: 夜泣き、イヤイヤ、きょうだい、自分の疲れ
   - トーン: 責めない・断定しない

3. **できること（3〜4点）**
   - うちの子に合う相談（覚えながら）
   - 週次の振り返り（Plus）
   - 複数の子・関係（Plus / 体験期間）
   - あなた（保護者）の相談（Plus / 体験期間）
   - **医療ではない** 一文（小さく、目立ちすぎない）

4. **プラン比較（簡易表）**

   | | 初回14日 | Free（体験後） | Plus |
   |---|---|---|---|
   | 価格 | 無料 | 無料 | ¥980/月 |
   | 1日の相談 | 無制限 | 5回 | 無制限 |
   | お子さん | 複数可 | 1人 | 複数可 |
   | あなた（保護者）の相談 | あり | なし | あり |
   | 週次レポート | あり | 過去分閲覧可 | あり |

   - 詳細は `plans.ts` と矛盾しないこと

5. **使い方（3ステップ）**
   - メールでログイン → お子さん登録 → 相談

6. **FAQ（5〜8問）**
   - 例は **6.2** 参照

7. **CTA（再掲）**
   - 無料ではじめる → `/login`

8. **フッター**
   - 法務リンク + コピーライト

#### 4.1.2 UX 要件

| 項目 | 要件 |
|------|------|
| デバイス | **モバイルファースト**（幅 375px 基準） |
| 操作 | 片手・親指で届く CTA（画面下部固定は任意） |
| 視覚 | 白〜薄青、余白多め、アニメーション控えめ |
| 文字量 | 1セクション 3〜5行まで。長文は法務ページへ |
| 性能 | LCP 3秒以内目標。画像は `logo.png` 中心、重い素材禁止 |

#### 4.1.3 コンテンツ禁止

- 「完治」「診断」「治療」など医療断定
- 過剰な限定・煽り（「今だけ！」連発）
- 他社の誹謗

### 4.2 利用規約（`/terms`）

#### 必須条項（章立て案）

1. 適用・サービス概要（育児支援 AI、医療行為ではない）
2. アカウント（メール OTP、16歳未満は保護者利用の想定）
3. 利用料金・プラン（Free / 初回14日 / Plus ¥980/月）
4. 禁止事項（違法行為、他者の権利侵害、システム妨害等）
5. コンテンツ・AI 応答（参考情報、正確性の非保証、緊急時は専門機関へ）
6. 知的財産
7. 免責・責任制限（法令の範囲内）
8. 解約・返金（**Stripe / マイページのお支払い管理** を案内。返金は運用ポリシーに沿う旨）
9. サービス変更・終了
10. 準拠法・管轄（日本法）
11. 改定（改定日・効力発生日をページ上部に表示）
12. お問い合わせ（`/contact` へのリンク）

#### クローラー・検索（確定）

| 手段 | 内容 |
|------|------|
| `metadata.robots` | `noindex, nofollow`（`src/app/legal/page.tsx`） |
| `X-Robots-Tag` | middleware で `noindex, nofollow, noarchive, nosnippet, noimageindex` |
| `robots.txt` | `User-agent: *` および GPTBot 等の **AI 向け UA** で `Disallow: /legal` |
| middleware | AI クローラーの User-Agent で `/legal` へのアクセスは **403** |

実装: `src/lib/crawlerPolicy.ts`, `src/app/robots.ts`, `middleware.ts`

#### 表示要件

- **最終更新日** をページ最上部に表示
- 読みやすい行間・見出し階層（h2/h3）
- PDF 不要（HTML のみで可）

### 4.3 プライバシーポリシー（`/privacy`）

#### 必須記載（個人情報保護法・実務）

1. 事業者名・連絡先（**6.1**。連絡は **`/contact` フォーム** — メール直書き不要）
2. 取得する情報
   - メールアドレス（認証）
   - お子さん・保護者プロフィール（名前、誕生日、性別任意、関係）
   - 相談内容・会話履歴
   - 課金情報（**Stripe が処理。カード番号は当方で保持しない**）
   - 利用ログ・トークン使用量（サーバー）
3. 利用目的（サービス提供、品質改善、課金、サポート）
4. 第三者提供（OpenAI API、Supabase、Stripe、Resend 等 — **サービス名を列挙**）
5. 国外への送信（OpenAI 等がある場合はその旨）
6. 保管期間・削除（アカウント削除手順があれば記載。なければ「お問い合わせ」）
7. ユーザーの権利（開示・訂正・削除の請求方法 → `/contact`）
8. Cookie・ローカルストレージ（認証セッション）
9. 改定

#### サービス固有の注意

- AI に送信される会話内容の取り扱いを **平易な日本語** で説明
- 子どもの情報を含むため、**保護者が登録・管理する** 前提を明記

### 4.4 特定商取引法に基づく表記（`/legal`）

#### 事業者情報を「控える」ことについて（方針）

**完全に非表示にはできない**（Plus など有料の通信販売を行う場合、特商法上の表示義務がある）。  
ただし次は可能で、**個人の自宅・本名を LP に直書きしない**運用も一般的です。

| 項目 | 公開 LP に載せる | 非公開にできる／代替 |
|------|------------------|----------------------|
| 販売事業者名（屋号） | **○ 必須** | 本名ではなく **屋号・サービス名** で可（例: となりっこ運営） |
| 運営責任者氏名 | **○ 必須** | 代表者名または通信販売担当者名 |
| 所在地 | **○ 必須** | **自宅以外**の住所（バーチャルオフィス・私書箱等、契約上の「所在地」） |
| 電話番号 | 原則表示 | **番号を載せず**「お問い合わせフォームより請求いただければ遅滞なく開示」等の記載で対応する運用が多い（要最新法令確認） |
| 連絡先 | **○ 必須** | **メールアドレスの公開は不要** → **`/contact` のフォーム**で可 |

※ 確定の法的判断は専門家へ。Stripe ダッシュボード上の事業者情報は別途必要。

#### 必須項目（電子契約・継続課金）

| 項目 | 記載内容（確定値） |
|------|-------------------|
| 販売事業者名 | **6.1 で確定**（屋号可） |
| 運営責任者 | 同上 |
| 所在地 | **6.1**（請求・開示可能な住所。自宅の直書きは避ける） |
| 電話番号 | **6.1**（開示請求対応の一文、または番号） |
| お問い合わせ | **`/contact` のフォーム**（メール直書きしない） |
| 販売URL | `https://www.tonarikko.com` |
| 販売価格 | Plus **月額 980円（税込）** — 価格変更時は改定 |
| 商品代金以外の必要料金 | インターネット接続料等はユーザー負担 |
| 支払方法 | クレジットカード（Stripe） |
| 支払時期 | 申込時および毎月の更新日 |
| 役務の提供時期 | 決済完了後、直ちに Plus 機能を利用可能 |
| 解約・返金 | マイページ「お支払い管理」から解約。返金はお問い合わせ（**6.3 運用方針**） |
| 動作環境 | スマートフォン・PC のモダンブラウザ（Safari / Chrome 等） |

### 4.5 お問い合わせ・サポート（`/contact`）

#### 方針（v1）

- **メールアドレスは LP に公開しない**
- **お問い合わせフォームを主手段**とする（Google フォーム / Tally / Notion フォーム / 自前 API のいずれか）

#### 必須要素

1. **お問い合わせフォーム**（必須）
   - 項目例: お名前（任意可） / メールアドレス（返信用・必須） / 種別（課金・不具合・個人情報・その他） / 本文
   - 送信後: 「受け付けました」画面 or 文言
   - スパム対策:  honeypot または CAPTCHA（必要に応じて）
   - 通知: フォーム送信先に **運営者メールへ通知**（ユーザーには通知先メールを公開しない）
2. **対応範囲**
   - アカウント・ログイン
   - 課金・解約（自己解約手順へのリンク: アプリ マイページ）
   - 不具合・要望
   - 個人情報の開示・削除請求
3. **返信目安**（例: 3営業日以内）
4. **緊急時の注意**
   - 育児の緊急・医療的判断は **119 / 専門機関** へ（AI は代替にならない）

#### 任意

- FAQ へのリンク（LP 内アンカー or 別セクション）

## 6. コンテンツ確定前のプレースホルダ（事業者記入）

### 6.1 事業者情報（法務・特商法・プライバシーで共通）

**公開方針:** 自宅・個人メールは載せない。屋号・事業用住所・フォームで連絡。

```
[×] 販売事業者名（屋号） … **「となりっこ」**（確定）
[—] 運営責任者名 … **一旦空欄**（`/legal` は `—` 表示。`src/features/marketing/legalContent.ts`）
[—] 所在地 … **一旦空欄**（同上。バーチャルオフィス確定後に填入）
[ ] 電話番号の扱い（番号を載せる / フォームからの開示請求に応じる旨を /legal に記載）
[ ] お問い合わせフォーム URL または埋め込み ID（Google Form 等）
[ ] フォーム通知先メール（非公開・運営者のみ）
```

**載せないもの（LP 上）**

- 個人用 `support@...` の直書き（フォーム経由のみ）
- 原則として自宅住所・私人の携帯番号

### 6.2 FAQ 案（LP 用）

| 質問 | 回答の方向性 |
|------|----------------|
| 無料で使えますか？ | 初回14日は Plus 同等。以降は Free（1日5回等） |
| 料金はいくらですか？ | Plus 月額980円。解約はいつでも |
| 医療的なアドバイスですか？ | いいえ。育児支援。不安なときは専門家へ |
| 子どもの情報は安全ですか？ | プライバシーポリシーへ。暗号化・第三者への送信範囲 |
| 解約方法は？ | マイページ → お支払い管理（Stripe Portal） |
| スマホに追加できますか？ | Safari からホーム画面に追加（手順は簡潔に） |
| 7歳以上でも使えますか？ | 利用可。0〜6歳向けに最適化と記載 |

### 6.3 返金・サポート運用方針（LP / 規約に反映）

- **解約:** ユーザーは Customer Portal から可能（即時または期末は Stripe 設定に依存）
- **返金:** 原則 Stripe Dashboard で個別対応。返金のみでは Plus が残る場合がある → 返金時は解約も行う旨を **運用側** で統一（`docs/OPERATIONS.md` 参照）

## 7. 技術要件

### 7.1 トップ `/` との関係（要決定）

| 案 | 説明 | 推奨 |
|----|------|------|
| A | `/` = LP、ログイン済みは `/chat` へリダイレクト | マーケ向き。変更量大 |
| B | `/` = 現状どおりチャット、LP は `/lp` のみ | **現状維持しつつ法務を追加するなら B** |
| C | `/` = LP、チャットは `/app` に移動 | 明確だが URL 変更の影響大 |

**本要件のデフォルト:** 案 **B**（`/lp` + 法務4ページを追加、トップは既存）

### 7.2 実装

| 項目 | 要件 |
|------|------|
| フレームワーク | 既存 Next.js App Router（`src/app/`） |
| スタイル | Tailwind。チャットと同系の青・白・グレー |
| コンポーネント | `src/components/` または `src/features/marketing/` にフッター・LP セクションを共通化 |
| SEO | 各ページ `metadata`（title / description）。OG は `brand.ts` 準拠 |
| アクセシビリティ | 見出し階層、リンクの明確なラベル、コントラスト |
| 多言語 | 日本語のみ（v1） |

### 7.3 パフォーマンス・法務

- 法務ページは **静的生成（SSG）可**
- 本文は Markdown または TSX 内定数。バージョン管理は Git
- 改定時は **最終更新日** を手動更新
